About Me
Engineer. Architect.
Builder.

I'm a Library & Solution Architect at Rockwell Automation, where I lead six device libraries within the Device Object Library (DOL) — a product used by industrial engineers worldwide to build standardized, scalable automation solutions across every major industry.

My core expertise is device-to-HMI/SCADA data integration — connecting power monitors, I/O modules, safety devices, network hardware, and IO-Link sensors to FactoryTalk Optix, FTView SE/ME, and PlantPAx in clean, reusable, and maintainable ways. I've done this across thousands of device objects over 12+ years, defining the architecture standards the whole team builds on.

Beyond my core role I'm deeply interested in AI agents and large language models. I build agentic workflows using the Anthropic API, experiment with tool-use patterns and multi-step reasoning chains, and follow developments across Claude, GPT-4, and Gemini. I believe AI-augmented engineering is the next frontier — and I'm actively positioning myself at that intersection.

I've also built four internal productivity tools from scratch in VB.NET — a graphics properties exporter, global object rename tool, smart quality checker, and project navigation report generator — tools actively used by the team today.

PowerMonitor 5000
PQEM Solution

Redesigned the power quality & energy monitoring faceplate for FactoryTalk Optix from the ground up. Integrates real-time data directly from PM5000 power monitors using the Rockwell Device API and Optix C# scripting — bypassing the PLC entirely. Delivers live dashboards for voltage, current, harmonics, power quality events, and energy trending with a clean, responsive HMI.

The first solution in the Rockwell ecosystem to achieve full PLC-independent operation for a power monitoring faceplate. Recognized by SVP LCS and presented to India Tech Community and GEBS leadership forums.
